160  static int findRegion(EGS_Float xp, int np, const EGS_Float *p) {
161  int ml = 0, mu = np;
162  while (mu - ml > 1) {
163  int mav = (ml+mu)/2;
164  if (xp <= p[mav]) {
165  mu = mav;
166  }
167  else {
168  ml = mav;
169  }
170  }

1129 /* \example geometry/example1/geometry_example1.cpp
1130 
1131  Suppose that you frequently use the same complex geometry and you
1132  are tired of always having to include the long definition of this geometry
1133  into your input file. In this case it may be worthwhile to implement your
1134  own geometry DSO that constructs the geometry of interest with very little
1135  input (or even no input at all). This example illustrates how to do this
1136  programatically.
1137 
1138  As this is just an example, the geometry we will be constructing is not very
1139  complex: it consists of a cylinder inscribed in a sphere and the sphere
1140  inscribed in a box as shown in the figure ???. We want to be able to define
1141  the media of the cylinder, the sphere and the box in the input file
1142  (so that we can easily use different PEGS data sets, for example). All other
1143  dimensions of our example geometry are fixed.
1144 
1145  To create our new geometry, we make a new subdirectory in the egspp
1146  geometries subdirectory called \c example1. To be able to use the
1147  predefined rules for building geometry libraries we create a dummy
1148  header file \c %geometry_example1.h that does nothing else except to
1149  define the \c EGS_EXAMPLE1_DLL
1150  macro:
1151  \include geometry/example1/geometry_example1.h
1152  and put the implementation in \c %geometry_example1.cpp.
1153  \dontinclude geometry/example1/geometry_example1.cpp
1154  In this file we include the just created header file and the EGS_Input
1155  class header file to get access to its definition,
1156  needed in the \c %createGeometry() function (see below):
1157  \until egs_input
1158  We also need the declarations of the various geometry classes that we
1159  will use to construct our geometry:
1160  \until egs_envelope_geometry.h
1161  We then define the dimensions of the various geometrical structures
1162  \until box_size
1163 
1164  The remaining task is to implement a C-style \c %createGeometry() function
1165  exported by the DSO
1166  \until EGS_EXAMPLE1_EXPORT
1167  We check that the input object is valid and return \c null if it is not
1168  \until }
1169  We then look for a definition of the cylinder medium and print a warning if
1170  such a definition is missing:
1171  \skipline cyl_medium
1172  \until }
1173  We repeat basically the same code to obtain the sphere and the box
1174  media
1175  \until }
1176  \until }
1177  and return \c null if any of the media definition was missing:
1178  \until if
1179  Now we are ready to construct our geometry. We first create a set of
1180  parallel z-planes
1181  \until EGS_ZProjector
1182  As the planes will only be used by our geometry object internally, we don't
1183  care how the plane set is named and therefore give it a name using
1184  the static function EGS_BaseGeometry::getUniqueName().
1185  In a similar way we create a set of z-cylinders consisting of a single
1186  cylinder
1187  \until ZProjector
1188  and make a closed cylinder modeled as a 2D-geometry from the planes and the
1189  cylinder surface:
1190  \until getUniqueName
1191  We then set the medium of the cylinder
1192  \until setMedium
1193  The next step is to create the sphere as a set of spheres consisting of
1194  a single sphere
1195  \until getUniqueName
1196  and to fill it with the sphere medium defined in the input
1197  \until setMedium
1198  We then use an envelope geometry to inscribe the cylinder into the just
1199  created sphere:
1200  \until getUniqueName
1201  The next step is to create the outer box and fill it with the box medium
1202  defined in the input
1203  \until setMedium
1204  (as our geometry will always be centered about the origin, we pass a
1205  \c null affine transformation to the constructor of the box).
1206  We now inscribe the \c the_sphere geometry object (which is the sphere
1207  with the cylinder inscribed in it) into the just created box using
1208  again an envelope geometry,
1209  \until the_box
1210  set the name of \c the_box from the input using the inherited
1211  \link EGS_BaseGeometry::setName(EGS_Input*) setName() \endlink method,
1212  \until setName
1213  and return a pointer to the just created geometry
1214  \until }
1215  \until }
1216  That's all.
1217 
1218  In the Makefile we include the egspp make config files,
1219  \dontinclude geometry/example1/Makefile
1220  \skipline EGS_CONFIG
1221  \until my_machine
1222  set the C-preprocessor defines to the standard set of defines
1223  for our egspp configuration,
1224  \until DEFS
1225  define the name of our DSO,
1226  \until library
1227  and and the set of files needed to build the DSO
1228  \until lib_files
1229  Because we are using planes, cylinders, spheres, a ND-geometry, a box
1230  geometry and an envelope geometry to construct our geometry, we must
1231  link against the geometry libraries containing these geometries.
1232  We accomplish this by seting the \c link2_libs make variable to the
1233  list of needed libraries
1234  \until egs_genvelope
1235  We then include the standard rules for building a DSO
1236  \until include
1237  and set the dependencies of our object files
1238  \until make_depend
1239  We must also add the dependencies on the various geometry header files
1240  that we are including
1241  \until egs_envelope_geometry.h
1242  We can now build our geometry DSO by typing \c make and use
1243  it by including input like this in the input file
1244  \verbatim
1245  :start geometry:
1246  library = geometry_example1
1247  name = some_name
1248  cylinder medium = H2O521ICRU
1249  sphere medium = AL521ICRU
1250  box medium = AIR521ICRU
1251  :stop geometry:
1252  \endverbatim
1253  It is worth noting that we could have constructed this geometry as the
1254  \link EGS_UnionGeometry union \endlink of the cylinder pointed to
1255  by \c the_cyl, the sphere pointed to by \c sphere and the
1256  box pointed to by \c box. However, such an implementation would be
1257  much slower at run time because for each invocation of a geometry
1258  method the union will have to interogate the geometry methods of all
1259  3 objects. In contrast, the envelope geometry will be checking only
1260  against the box for points outside of the box, the box and the sphere for
1261  points inside the box but outside the sphere, the sphere and the cylinder
1262  for points inside the sphere but outside the cylinder and the cylinder
1263  only for points inside the cylinder.
1264

1266  \include geometry/example1/Makefile
1267  \include geometry/example1/geometry_example1.h
